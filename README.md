## 一、项目概述

### 1.1 定位
轻量级个人项目进度追踪工具，帮助神冢记录自己做过的项目/功能点，打勾标记完成。

### 1.2 核心价值
- **可视化完成感**：打勾有动效，增强成就感
- **快速回顾**：知道自己做了什么、还剩什么
- **极简优先**：本地存储，够用就好

---

## 二、功能规划

### 2.1 核心功能

| # | 功能 | 优先级 | 状态 |
|---|------|--------|------|
| 1 | 新建项目（名称 + 链接） | P0 | 待开发 |
| 2 | 给项目添加任务（功能点） | P0 | 待开发 |
| 3 | 做完一个打勾（✅有动效） | P0 | 待开发 |
| 4 | 显示完成进度（3/5 完成 60%） | P0 | 待开发 |
| 5 | 按项目分组 / 按完成状态筛选 | P1 | 待开发 |
| 6 | Dark Mode | P1 | 待开发 |

### 2.2 交互细节

**新建项目**
- 输入：项目名称、项目链接（可选）
- 行为：创建后自动展开，显示任务输入框

**添加任务**
- 输入：任务描述
- 行为：添加到当前项目列表

**打勾完成**
- 行为：点击checkbox → 触发完成动效 → 进度更新
- 动效：打勾时有视觉反馈（放大/缩小/颜色变化）

**进度显示**
- 格式：「3/5 完成 60%」
- 计算：已完成任务数 / 总任务数

**筛选功能**
- 按项目分组查看
- 按完成状态筛选（全部 / 已完成 / 未完成）

## 三、UI设计要求（⚠️ 最高优先级）

### 3.1 设计标准
**「非常好看」** — 这是神冢的原话，设计要精致、有质感、经得起细看。

### 3.2 设计方向
- **风格**：参考 Linear / Raycast 的设计语言 — 深色、简洁、有层次感
- **配色**：深色背景 + 柔和渐变 + 高对比度强调色
- **字体**：等宽字体用于数字/进度，信息层次分明
- **动效**：流畅、克制、有完成感（打勾动效要让人有成就感）
- **间距**：宽松留白，呼吸感强

### 3.3 必做项
- [ ] 像素级完美的深色主题
- [ ] 打勾动效要精心调教（缓动曲线、时长、缩放比例）
- [ ] 进度条要有渐变和光泽感
- [ ] 项目卡片有微妙的边框/阴影层次
- [ ] 整体视觉要经得起截图发朋友圈

## 四、技术规划

### 4.1 技术栈

| 类别 | 工具 | 费用 | 说明 |
|------|------|------|------|
| AI编码 | Claude | $20/月 | 代码生成 |
| 部署 | Vercel | FREE | 免费托管 |
| 前端 | HTML/CSS/JS | - | 极简够用 |
| 存储 | localStorage | FREE | 本地存储够用 |

### 4.2 为什么这样选

- **Vercel**：免费、部署快、随时能打开
- **localStorage**：数据量小，本地足够，不需要后端
- **纯前端**：功能简单，不需要复杂后端架构
