## 1. 项目概述

### 1.1 项目背景

"羊了个羊"自2022年在中国市场爆红，以极低通关率（~0.1%）和社交分享机制创造了现象级传播。其核心玩法"两层三消"极具黏性，但截至2026年初，海外市场尚无同等量级的竞品。

中东阿拉伯国家（阿联酋、沙特、卡塔尔、科威特等）智能手机渗透率超过75%，WhatsApp是最主流的通讯工具（渗透率>90%），游戏玩家以18-35岁男性为主，对"挑战性"和"炫耀性"内容接受度极高。我们选择以"骆驼、猎鹰、钻石、棕榈树、油井、古代罐子、皇冠"等中东特色emoji为方块，打造本地化版本。

### 1.2 项目目标

| 目标 | 指标 |
|------|------|
| 上线首月UV | 50万 |
| 30日留存 | ≥15% |
| 激励广告ARPU | ≥$0.15 |
| 付费率 | ≥1.5% |
| 病毒系数（K因子） | ≥1.2 |

### 1.3 核心亮点

- **零门槛H5游戏**：无需下载，WhatsApp内置浏览器直接玩
- **超低通关率**：0.1%通过率制造稀缺话题性
- **强社交传播**：一键分享通关截图至WhatsApp群聊
- **中东特色主题**：7种本地化emoji方块，文化认同感强
- **双货币体系**：金币（免费产出）+ AED内购，支持激励广告减免

1. **无聊等待场景多**：中东男性有大量等待时间（等人、等餐、等会议）

## 3. 游戏规则详细说明

### 3.1 核心机制

**两层结构**：
- **底层（Bottom Layer）**：游戏开始时全部方块平铺在此层，被上层方块遮挡时不可点击
- **顶层（Top Layer）**：方块堆叠在底层上方，遮挡底层部分方块；顶层方块被消除后露出底层方块

**消除规则**：
- 每次从顶部可见方块中点击两个**图案相同**的方块即可消除
- 方块被消除后，其下方的方块变为可见状态
- 消除全部方块 = 通关；步数耗尽且无法消除 = 失败

**遮挡逻辑**：
- 方块A"遮挡"方块B的定义：B的上、下、左、右任一侧有A存在，且B不在最顶层
- 只有完全不受任何方块遮挡的方块可以被点击
- 检测方式：遍历所有方块，对每个方块检查四边是否有相邻方块在其上方层

### 3.2 步数系统

- 每个关卡开始时分配固定步数（例：Level 1=30步，Level 30=18步）
- 每次点击一个方块（无论是否成功配对）消耗1步
- 成功消除一对方块：+0步（不额外奖励步数，维持紧张感）
- 道具"加3步"：不额外增加步数上限，直接增加步数

### 3.3 道具系统

| 道具 | 图标 | 功能 | 获取方式 |
|------|------|------|----------|
| 撤回（Undo） | ↩️ | 撤销最后一次操作，恢复2个方块，返还1步 | 看广告 / 50金币 |
| 洗牌（Shuffle） | 🔀 | 随机打乱所有剩余方块的位置 | 看广告 / 80金币 |
| 复活（Revive） | ❤️‍🔥 | 失败后使用，返还3步并重新获得一次机会 | 看广告 / 200金币 |

### 3.4 通关条件

- **胜利条件**：在步数耗尽前消除所有方块
- **失败条件**：步数归零时仍有剩余方块
- **通关率设计**：Level 1-10约10-30%，Level 11-20约2-5%，Level 21-30约0.1-0.5%

## 4. 关卡设计规格

### 4.1 关卡结构

每个关卡包含：
- **关卡编号**（1-30）
- **难度星级**（1-3星，3星最难）
- **方块总数**（= 7种方块 × 每种方块数量，7的倍数）
- **步数上限**
- **上层阻挡物数量**（顶层方块数）

async function shareToWhatsApp(imageDataUrl, text) {
  if (navigator.share) {
    const blob = await (await fetch(imageDataUrl)).blob();
    const file = new File([blob], 'sheep-match.png', { type: 'image/png' });

      title: 'Sheep Match',

## 6. 激励广告场景设计

### 6.1 广告触发点

| 场景 | 广告类型 | 奖励 | 用户收益 |
|------|----------|------|----------|
| 首次失败 | 激励视频 | 复活+3步 | 续命重玩 |
| 每日首次 | 激励视频 | 随机道具×1 | 免费道具 |
| 道具耗尽 | 激励视频 | 道具×1 | 继续游戏 |
| 双倍金币关 | 激励视频 | 金币×2 | 加速积累 |
| 关卡通过 | 激励视频 | +5步 | 提高通关率 |

### 6.2 广告频次控制

- **每日激励广告上限**：10次/日
- **两次广告间隔**：≥30秒冷却
- **AdMob配置**：`adUnitType: 'rewarded'`，reward-based video ad

### 6.3 广告代码示例（Google AdMob）

```javascript
// reward-based video ad
let rewardedAd = null;

async function loadRewardedAd() {
  rewardedAd = new google.ads.api.rewarded.RewardedAd(clientId, {
    adUnitId: 'ca-app-pub-xxxxx/rewarded-unit-id'
  });
  const loader = new google.ads.api.rewarded.RewardedAdLoader({
    clientId, adUnitId
  });
  rewardedAd = await loader.load();
}

function showRewardedAd(rewardType) {
  if (!rewardedAd) return;
  rewardedAd.show({
    userId: getUserId(),
    customData: { rewardType }
  }).then(result => {
    const reward = result['reward'];
    grantReward(rewardType, reward.amount);
  });
}
```

## 8. 文化合规要求

### 8.1 视觉合规

- ❌ 不得出现猪、酒杯（伊斯兰禁忌）
- ❌ 不得出现赌博元素（老虎机、骰子主题）
- ❌ 不得出现以色列国旗、巴勒斯坦旗帜相关符号
- ❌ 不得出现偶像崇拜（明星照片、人物肖像）
- ✅ 使用本地文化符号：骆驼、猎鹰、沙漠、棕榈树、古代文明元素

### 8.2 音效合规

- ✅ 使用传统中东风格背景音乐（需确认版权）
- ✅ 音效简洁明快，无宗教内容
- ❌ 避免使用人类声音哼唱的歌曲（涉及文化敏感性）

### 8.3 文案合规

- ✅ 使用中性友好的挑战文案（不涉及宗教、政治）
- ✅ 阿拉伯语版本需RTL（从右至左）排版
- ✅ 避免"猪""酒""赌"等敏感词汇（中英阿三语）

### 8.4 数据合规

- ✅ 符合 UAE PDPL（个人数据保护法）要求
- ✅ 不存储用户WhatsApp联系人信息
- ✅ 游戏数据仅存储于用户本地（localStorage）
