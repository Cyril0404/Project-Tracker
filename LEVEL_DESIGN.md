## 1. 关卡设计原则

### 1.1 核心设计哲学

| 原则 | 说明 |
|------|------|
| **难度渐进** | 每个关卡的通过率应略低于前一关，形成自然的难度曲线 |
| **视觉引导** | 布局设计应让玩家"看见"关键方块的位置，而非随机猜测 |
| **策略深度** | 高难度关卡应有多条通关路径，而非只有唯一解 |
| **心理激励** | 每关失败都让玩家觉得"差一点就过了"，激发重试欲望 |
| **零重复感** | 30个关卡各有独特布局名称和视觉主题 |

### 1.2 布局类型分类

| 布局类型 | 描述 | 难度特征 |
|----------|------|----------|
| `standard` | 标准矩阵 | 简单，适合入门 |
| `stepped` | 阶梯式 | 中等，需规划路径 |
| `cross` | 十字交叉 | 中等，边角方块多 |
| `diamond` | 菱形 | 中等，中心是关键 |
| `tower` | 塔楼 | 中高，层层遮挡 |
| `chaos` | 散乱式 | 难，随机感强 |
| `bridge` | 桥梁 | 中，通道狭窄 |
| `wave` | 波浪式 | 中高，波峰遮挡 |
| `castle` | 城堡式 | 高，密集堆叠 |
| `pyramid` | 金字塔 | 高，层层递减 |
| `gauntlet` | 终极挑战 | 极难 |

### 1.3 难度参数计算

```javascript
function calculateDifficulty(level) {
  const baseDifficulty = level; // 1-30

  // 步数效率：方块总数/步数，越小越难
  const efficiency = totalBlocks / steps;

  // 层叠比：顶层方块数/底层方块数，越高越难
  const layerRatio = topLayerCount / bottomLayerCount;

  // 障碍指数：被遮挡方块比例
  const blockageRatio = blockedBlocks / totalBlocks;

  // 综合难度分
  const difficultyScore =
    baseDifficulty * 0.4 +
    efficiency * 10 * 0.3 +
    layerRatio * 5 * 0.2 +
    blockageRatio * 10 * 0.1;

  return difficultyScore;
}
```

### Level 1 — Desert Dawn ⭐
**通过率目标**: 28% | **步数**: 32 | **方块**: 28

**设计理念**：纯新手引导，关卡简洁，路径清晰，让玩家感受成就感。

**底层布局**（4行×7列）：
```
■ ■ ■ ■ ■ ■ ■   Row 0
■ ■ ■ ■ ■ ■ ■   Row 1
■ ■ ■ ■ ■ ■ ■   Row 2
■ ■ ■ ■ ■ ■ ■   Row 3
```
**顶层布局**（2行×4列，叠加在底层上方）：
```
■ ■ ■ ■   Row 0 (col 1-4)
■ ■ ■ ■   Row 1 (col 1-4)
```

**布局说明**：顶层4列方块正好遮挡底层中间区域，形成自然的"缺口"，玩家从两侧边缘开始消除即可。

**通关技巧**：先消除顶层方块，让底层方块露出来；优先匹配边缘方块。

## 4. 关卡设计Checklist

每个关卡上线前需检查：

- [ ] 方块总数是7的倍数（保证可以完全消除）
- [ ] 顶层方块数不超过底层方块数的70%
- [ ] 通过模拟器验证：随机打乱20次，是否有至少3次可以在给定步数内通关
- [ ] 步数设置：保证在无道具情况下有理论通关可能（但概率极低）
- [ ] 所有emoji在不同平台上显示一致（iOS/Android测试）
- [ ] 遮挡关系正确（底层被遮挡时不可点击）
- [ ] 通关率符合目标区间
